> Considere o cenário da "Operação Cadê o Equipamento?". Sabendo que o objetivo final é construir uma API segura, sua tarefa inicial é estruturar a base de dados que sustentará a lógica do sistema. 
---
A partir das regras de negócio do cenário, analise e defina:

1. Entidades Principais: Identifique as tabelas necessárias para que o sistema funcione. Certifique-se de contemplar a separação de papéis de usuários (RBAC), o catálogo de equipamentos e o registro de transações.

2. Dicionário de Dados (Campos): Especifique os campos essenciais para cada entidade. Destaque as chaves primárias (PK), chaves estrangeiras (FK) e os campos de controle de estado (ex: status_equipamento, status_solicitacao).

3. Modelagem Lógica (DER): Desenhe o modelo relacional que atenda às necessidades do sistema. O seu modelo deve provar que é capaz de responder a perguntas críticas, como: "Qual discente está com o notebook de testes de Linux neste exato momento?" e "Quem foi o gestor que autorizou esse empréstimo?".

Encaminhe via e-mail o modelo em uma imagem de alta resolução ou o link para visualização online através de ferramentas de modelagem (sugestões: brModelo, dbdiagram.io, Draw.io, Miro ou Figma). Certifique-se de que as cardinalidades (1:N, N:M) estejam visíveis e corretas.
